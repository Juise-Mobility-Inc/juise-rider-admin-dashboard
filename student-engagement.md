# Engage students

## Send a notification

Open **Notifications** to send a customer-visible push notification.

1. Select the intended audience.
2. Enter a concise title and message.
3. Add a supported destination link or image only when needed.
4. Review the preview and recipient scope.
5. Select **Send Notification** once.
6. Check notification history for the result.

Sending is immediate, not a draft workflow, and a message generally cannot be recalled. A sent result means the notification provider accepted the request; it does not guarantee that every device displayed it.

## Create a ride challenge

Use **Ride Challenges** for time-limited goals based on supported ride metrics.

1. Enter a clear student-facing title and description.
2. Choose the metric and target value.
3. Set the start and end times.
4. Confirm the intended audience and active status.
5. Add campaign artwork if needed.
6. Review the complete challenge before creating it.

Upcoming challenges have not started, live challenges are currently available, and ended challenges are past their end time. Download participant progress before deleting a challenge whose records you need to retain.

## Create a challenge game

Use **Challenge Games** for scavenger-hunt-style experiences based on GPS check-in stops. Each stop can include coordinates, a check-in radius, a clue, prize points, and a sequence.

Visit or independently verify every location before publishing. Ensure stops are safe, accessible, permitted by the property owner, and described without exposing sensitive locations.
